# 2.6 Inprimaki Sarreren Lotura (Form Input Bindings)

## v-model sarrera

`v-model`-ek formulario-elementu baten balioa eta JavaScript aldagaia automatikoki bi noranzkotan lotzen ditu. `:value` + `@input` bezalako lotura eskuzkoak sinplifikatzen ditu.

Elementu motaren arabera funtzionatzen du:

- **Testu-sarrera / Textarea**: `value` + `input`.
- **Checkbox / Radio**: `checked` + `change`.
- **Select**: `value` + `change`.

Ohar garrantzitsua: HTML hasierako balioak (`value`, `checked`, `selected`) ez dira kontuan hartzen; JS egoera da egia-iturri bakarra. Hasieratu JS-n (`ref`, `reactive`).

---

## Oinarrizko elementuekin erabilera

### Testu-sarrera

```html
<input v-model="message" placeholder="edit me" />
<p>Message: {{ message }}</p>
```

IME duten hizkuntzetarako (txinera, japoniera), erabil ezazu `compositionend` entzulea edo `.lazy`.

### Textarea (lerro anitz)

```html
<textarea v-model="message"></textarea>
<p style="white-space: pre-line;">{{ message }}</p>
```

Ez erabili `{{ }}` interpolazioa `<textarea>` barruan.

### Checkbox

- Booleanoa (bai/ez):

```html
<input type="checkbox" v-model="checked" />
<p>Checked: {{ checked }}</p>
```

- Hainbat aukera (array): balioen zerrenda bati lotzen zaio.

```html
<label><input type="checkbox" value="Jack" v-model="checkedNames" /> Jack</label>
<label><input type="checkbox" value="John" v-model="checkedNames" /> John</label>
<label><input type="checkbox" value="Mike" v-model="checkedNames" /> Mike</label>
<p>Hautatuak: {{ checkedNames.join(', ') }}</p>
```

### Radio (aukera bakarra)

```html
<label><input type="radio" value="One" v-model="picked" /> One</label>
<label><input type="radio" value="Two" v-model="picked" /> Two</label>
<p>Aukeratua: {{ picked }}</p>
```

### Select

- Sinplea:

```html
<select v-model="selected">
  <option disabled value="">Aukeratu aukera bat</option>
  <option>A</option>
  <option>B</option>
  <option>C</option>
</select>
<p>Hautatua: {{ selected }}</p>
```

- Anitza: `multiple` eta array-a erabili.

```html
<select v-model="selectedMany" multiple size="4">
  <option value="a">A</option>
  <option value="b">B</option>
  <option value="c">C</option>
  <option value="d">D</option>
</select>
<p>Hautatuak: {{ selectedMany.join(', ') }}</p>
```

- Dinamikoa (`v-for` eta `{ text, value }` objektuekin). Gomendioa: gehitu hasierako aukera huts eta desgaitua (adib. iOS bateragarritasunerako).

```html
<select v-model="country">
  <option disabled value="">Hautatu herrialdea</option>
  <option v-for="opt in countries" :key="opt.value" :value="opt.value">
    {{ opt.text }}
  </option>
</select>
```

```js
import { ref } from 'vue'
export default {
  setup() {
    const countries = ref([
      { text: 'Espainia', value: 'es' },
      { text: 'Frantzia', value: 'fr' },
      { text: 'Italia', value: 'it' }
    ])
    const country = ref('')
    return { countries, country }
  }
}
```

## Balio-loturak (value bindings)

- Checkbox balio pertsonalizatuekin: `true-value` eta `false-value`.

```html
<input type="checkbox" v-model="agree" true-value="yes" false-value="no" />
<p>Ados: {{ agree }}</p>
```

- Radio dinamikoa: `:value="first"`.

```html
<input type="radio" :value="first" v-model="pickedObj" />
```

- Select objektuekin: aukera balio gisa objektua lotu.

```html
<select v-model="selectedUser">
  <option :value="{ id: 1, name: 'Ada' }">Ada</option>
  <option :value="{ id: 2, name: 'Linus' }">Linus</option>
</select>
<pre>{{ selectedUser }}</pre>
```

Oharra: objektuen konparazioa erreferentziaren arabera egiten da; erabili gako egonkorrak edo normalizatu.

---

## v-model aldagailuak (modifiers)

- `.lazy`: `change` gertaeran eguneratzen du (`input` ordez).
- `.number`: `parseFloat()` bidez bihurtzen du zenbakira (automatikoki `type="number"`-en).
- `.trim`: zuriune hasiera/amaierak kentzen ditu.

```html
<input v-model.lazy="msg" />
<input type="number" v-model.number="age" />
<input v-model.trim="query" />
```

---

## v-model osagai pertsonalizatuetan

`v-model` osagai berrerabilgarrietan ere erabil daiteke. `modelValue` prop-a eta `update:modelValue` ekitaldia behar dira. Xehetasunak: osagaien gidako `v-model` atalean.
